常用指令

# 容器相关指令

# 1. docker run

  • 启动一个新的容器并运行指定的镜像。
  • 语法：docker run [OPTIONS] <image_name>
  • 例子:
docker run -d -p 8080:80 nginx:latest

解释：

  • -d：以分离模式启动容器，容器将在后台运行。
  • -p 8080:80：将宿主机的 8080 端口映射到容器的 80 端口。
  • nginx:latest：使用 nginx 镜像的 latest 标签启动容器。

# 2. docker ps

  • 列出正在运行的容器。
  • 语法：docker ps [OPTIONS]
  • 例子:
docker ps -a --filter "status=exited"

解释：

  • -a：列出所有容器（包括已经停止的容器）。
  • --filter "status=exited"：筛选出状态为 "exited" 的容器。

# 3. docker exec

  • 在运行的容器中执行命令。
  • 语法：docker exec [OPTIONS] <container_name> <command>
  • 例子:
docker exec -it mycontainer bash

解释：

  • -it：-i 保持标准输入流，-t 分配伪终端，允许交互式命令执行。
  • mycontainer：指定要执行命令的容器名称。
  • bash：在容器内启动一个 Bash shell。

# 4. docker stop

  • 停止一个运行中的容器。
  • 语法：docker stop <container_name>
  • 例子:
docker stop mycontainer

解释：

  • mycontainer：指定要停止的容器名称。

# 5. docker start

  • 启动一个停止的容器。
  • 语法：docker start <container_name>
  • 例子:
docker start mycontainer

解释：

  • mycontainer：指定要启动的容器名称。

# 6. docker restart

  • 重启一个运行中的容器。
  • 语法：docker restart [OPTIONS] <container_name>
  • 例子:
docker restart mycontainer

解释：

  • mycontainer：指定要重启的容器名称。

# 7. docker rm

  • 删除一个容器。
  • 语法：docker rm [OPTIONS] <container_name>
  • 例子:
docker rm mycontainer

解释：

  • mycontainer：指定要删除的容器名称。

# 8. docker logs

  • 查看容器的日志。
  • 语法：docker logs [OPTIONS] <container_name>
  • 例子:
docker logs -f --tail 100 mycontainer

解释：

  • -f：实时跟踪日志输出。
  • --tail 100：显示日志的最后 100 行。
  • mycontainer：指定要查看日志的容器名称。

# 镜像相关指令

# 1. docker pull

  • 从 Docker 仓库拉取指定镜像。
  • 语法：docker pull [OPTIONS] <image_name>
  • 例子:
docker pull nginx:latest

解释：

  • nginx:latest：拉取 nginx 镜像的 latest 标签。

# 2. docker build

  • 使用 Dockerfile 构建一个镜像。
  • 语法：docker build [OPTIONS] <context>
  • 例子：
docker build -t myapp:1.0 .

解释：

  • -t myapp:1.0：为镜像指定标签 myapp:1.0。
  • .：构建上下文为当前目录，包含 Dockerfile 文件。

# 3. docker images

  • 列出本地已有的镜像。
  • 语法：docker images [OPTIONS]
  • 例子:
docker images --filter "dangling=true"

解释：

  • --filter "dangling=true"：筛选出悬空镜像（没有标签的镜像）。

# 4. docker rmi

  • 删除本地的镜像。
  • 语法：docker rmi [OPTIONS] <image_name>
  • 例子:
docker rmi myapp:1.0

解释：

  • myapp:1.0：删除标签为 myapp:1.0 的镜像。

# 5. docker tag

  • 为本地的镜像添加标签。
  • 语法：docker tag <source_image> <target_image>
  • 例子:
docker tag myapp:1.0 myapp:latest

解释：

  • myapp:1.0：源镜像的标签。
  • myapp:latest：目标镜像的标签。

# 网络管理相关指令

# 1. docker network ls

  • 列出 Docker 主机上所有的网络。
  • 语法：docker network ls [OPTIONS]
  • 例子:
docker network ls --filter "driver=bridge"

解释：

  • --filter "driver=bridge"：筛选出使用 bridge 驱动的网络。

# 2. docker network inspect

  • 查看 Docker 主机上指定的网络的详细信息。
  • 语法：docker network inspect <network_name>
  • 例子:
docker network inspect bridge

解释：

  • bridge：查看默认的桥接网络 bridge 的详细信息。

# 3. docker network create

  • 创建一个新的 Docker 网络。
  • 语法：docker network create [OPTIONS] <network_name>
  • 例子:
docker network create --driver bridge my_network

解释：

  • --driver bridge：指定使用 bridge 网络驱动。
  • my_network：新创建的网络名称。

# 卷管理相关指令

# 1. docker volume ls

  • 列出所有 Docker 卷。
  • 语法：docker volume ls [OPTIONS]
  • 例子:
docker volume ls --filter "driver=local"

解释：

  • --filter "driver=local"：筛选出使用 local 驱动的卷。

# 2. docker volume inspect

  • 查看指定卷的详细信息。
  • 语法：docker volume inspect <volume_name>
  • 例子:
docker volume inspect my_volume

解释：

  • my_volume：查看名为 my_volume 的卷的详细信息。

# 3. docker volume create

  • 创建一个新的 Docker 卷。
  • 语法：docker volume create [OPTIONS] <volume_name>
  • 例子:
docker volume create --name my_volume

解释：

  • --name my_volume：指定新创建的卷的名称为 my_volume。

# 4. docker volume rm

  • 删除一个 Docker 卷。
  • 语法：docker volume rm <volume_name>
  • 例子:
docker volume rm my_volume

解释：

  • my_volume：删除名为 my_volume 的卷。

# 卷的常见用法

# 1. 持久化数据库数据

  • 容器中运行的数据库（如 MySQL、PostgreSQL）通常会使用卷来存储数据文件，这样即使容器重启或销毁，数据仍然保留。
docker run -v my_db_volume:/var/lib/mysql mysql

解释：

  • -v my_db_volume:/var/lib/mysql：此命令将一个名为 my_db_volume 的 Docker 卷挂载到容器内的 /var/lib/mysql 目录。MySQL 数据库的数据文件默认存储在该目录下。通过将 MySQL 数据库的数据目录挂载到卷上，即使容器重启或被删除，数据库中的数据依然会保存在 my_db_volume 卷中。
    • my_db_volume 是一个 Docker 卷的名称。卷会由 Docker 自动创建并管理。
    • /var/lib/mysql 是 MySQL 容器内的数据存储路径，通常这个目录包含了数据库的数据文件。

# 2. 共享文件

  • 不同容器之间需要共享数据时，可以使用卷将数据挂载到多个容器中。
docker run -v shared_volume:/data my_container

解释：

  • -v shared_volume:/data：这个命令将名为 shared_volume 的卷挂载到容器内的 /data 目录。通过挂载卷，容器内的数据就可以保存到这个卷中。
    • shared_volume 是本地 Docker 卷的名称。
    • /data 是容器内的目录，容器将会在这个目录中访问和存储数据。

# 3. 存储日志文件

  • 日志文件通常需要持久化，并可能需要在宿主机上查看。通过卷，容器可以将日志文件输出到宿主机目录。
docker run -v /host/logs:/container/logs nginx

解释：

  • -v /host/logs:/container/logs：将宿主机的 /host/logs 目录挂载到容器的 /container/logs 目录。容器中的应用（如 Nginx）可能会在 /container/logs 目录中生成日志文件，通过挂载卷，我们可以将日志文件保存在宿主机的 /host/logs 目录中。
    • /host/logs 是宿主机上的目录，用来存储日志文件。
    • /container/logs 是容器内的日志文件目录。

# 4. 备份数据

  • 卷的持久化特性使得它非常适合用于容器的数据备份。可以通过将卷中的数据复制到宿主机上或其他地方来进行备份。
docker run --rm -v my_volume:/data -v /host/backup:/backup alpine cp -r /data /backup

解释：

  • --rm：表示容器在执行完命令后会自动删除，不会保留容器。
  • -v my_volume:/data：将名为 my_volume 的卷挂载到容器内的 /data 目录。my_volume 卷可能存储了一些需要备份的数据。
  • -v /host/backup:/backup：将宿主机的 /host/backup 目录挂载到容器内的 /backup 目录。容器内的 /backup 目录用来存储备份的数据。
  • alpine cp -r /data /backup：在容器内运行 cp 命令，将 /data 目录中的所有内容复制到 /backup 目录，-r 选项表示递归复制所有子目录和文件。

# Docker Compose 相关指令

# 1. docker-compose up

  • 启动并运行 Docker Compose 定义的服务。
  • 语法：docker-compose up [OPTIONS]
  • 例子:
docker-compose up -d --build

解释：

  • -d：以分离模式启动容器，即在后台运行。
  • --build：在启动服务前重新构建镜像。

# 2. docker-compose down

  • 停止并删除 Docker Compose 启动的容器、网络等资源。
  • 语法：docker-compose down [OPTIONS]
  • 例子:
docker-compose down --volumes

解释：

  • --volumes：同时删除与服务相关的卷。

# 3. docker-compose logs

  • 查看 Docker Compose 启动的服务的日志。
  • 语法：docker-compose logs [OPTIONS] [SERVICE...]
  • 例子:
docker-compose logs -f --tail 100

解释：

  • -f：实时跟踪日志输出。
  • --tail 100：显示日志的最后 100 行。

# 4. docker-compose build

  • 构建 Docker Compose 定义的服务镜像。
  • 语法：docker-compose build [OPTIONS] [SERVICE...]
  • 例子:
docker-compose build --no-cache

解释：

  • --no-cache：构建过程中不使用缓存。
