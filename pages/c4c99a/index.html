常用指令

# 1. 内存管理参数

参数 示例 说明
-Xms<size> -Xms512m 设置 JVM 初始堆大小为 512MB。
-Xmx<size> -Xmx1024m 设置 JVM 最大堆大小为 1024MB，堆内存的上限，超过此值会抛出 OutOfMemoryError。
-Xmn<size> -Xmn256m 设置年轻代大小为 256MB，年轻代负责短生命周期对象的内存分配与回收。
-XX:MetaspaceSize=<size> -XX:MetaspaceSize=128m 设置元空间的初始大小为 128MB（JDK 8 及以上）。
-XX:MaxMetaspaceSize=<size> -XX:MaxMetaspaceSize=256m 设置元空间的最大大小为 256MB，元空间存放类的元数据。
-XX:HeapDumpPath=<path> -XX:HeapDumpPath=/logs/dump 设置堆转储文件生成的路径为 /logs/dump，通常用于内存泄漏分析。

# 2. 垃圾回收器参数

参数 示例 说明
-XX:+UseSerialGC 使用串行垃圾回收器，适合单线程应用程序，使用简单但效率低。
-XX:+UseParallelGC 使用并行垃圾回收器，适合多核 CPU，优化吞吐量。
-XX:+UseG1GC 使用 G1 垃圾回收器，适用于低延迟、大内存的应用场景，减少垃圾回收停顿时间。
-XX:MaxGCPauseMillis=<n> -XX:MaxGCPauseMillis=200 设置 GC 最大停顿时间为 200 毫秒（适用于 G1 垃圾回收器）
-XX:+PrintGCDetails 打印详细的垃圾回收日志信息，包括每次 GC 的时间、类型和内存变化。
-XX:+PrintGCDateStamps 在 GC 日志中增加时间戳，用于排查历史日志问题。
-XX:+HeapDumpOnOutOfMemoryError 当发生 OutOfMemoryError 时生成堆转储文件，用于分析内存问题

# 3. 线程相关参数

参数 示例 说明
-Xss<size> -Xss1m 设置每个线程的栈大小为 1MB。较大的栈大小可以支持更深的递归调用，但会占用更多内存。默认值通常是 512KB 或 1MB。

# 4.性能优化相关参数

参数 示例 说明
-Xverify:none 禁用字节码验证，减少类加载的开销，提高启动速度。适合在生产环境下的优化，但需确保代码无安全风险。
-XX:+AggressiveOpts 启用试验性优化参数，可能带来性能提升，但功能尚未完全稳定。通常用于测试环境或性能测试阶段。
-XX:+UnlockExperimentalVMOptions 解锁 JVM 的实验性参数。与 -XX:+UseG1GC 等选项搭配使用时需要此参数。
-XX:CompileThreshold=<n> -XX:CompileThreshold=1000 设置方法的调用次数达到 1000 次后触发 JIT 编译。适用于需要频繁调用的小方法优化。

# 5. 示例组合

# 使用 OpenJDK 作为基础镜像
FROM openjdk:17-jdk-slim

# 设置工作目录为 /app/fmea
WORKDIR /app/fmea

# 将 target 复制到容器中的 /app/fmea
COPY target /app/fmea

# 设置类路径，包含 config、lib 和 JAR 文件
ENV CLASSPATH=/app/fmea/config:/app/fmea/lib/*:/app/fmea/lib/palmdoit_server-0.0.1-SNAPSHOT.jar

# JVM 调优参数
ENV JAVA_OPTS="-Xms512m -Xmx1024m -XX:+UseG1GC -XX:MaxGCPauseMillis=200 -XX:+HeapDumpOnOutOfMemoryError -XX:HeapDumpPath=/app/fmea/logs/heapdump.hprof"

# 暴露端口
EXPOSE 7007

# 设置容器启动命令，启动主类，并使用 JAVA_OPTS
ENTRYPOINT ["sh", "-c", "java $JAVA_OPTS -Xverify:none -cp /app/fmea/config:/app/fmea/lib/*:/app/fmea/lib/palmdoit_server-0.0.1-SNAPSHOT.jar com.palmdoit.common.MyConfig"]
